Workflow per lavorare sul progetto con branch ale

1 — Prima di iniziare a lavorare

Ogni volta che apri il progetto, aggiorna il tuo branch con le ultime modifiche da main:

# 1. Vai su main e scarica le novità
git checkout main
git pull origin main

# 2. Torna su ale e porta le novità
git checkout ale
git merge main
Questo assicura che il tuo branch sia sempre aggiornato prima di iniziare.

2 — Avvia il sito in locale

Per vedere il sito mentre lavori:

npm run dev

Poi apri il browser su http://localhost:3000

3 — Mentre lavori

Modifica i file normalmente. Le modifiche si riflettono in tempo reale sul browser. Quando sei soddisfatto di una modifica, salvala su GitHub:

# Aggiungi tutti i file modificati
git add .

# Crea il commit con una descrizione
git commit -m "descrizione di cosa hai fatto"

# Manda su GitHub
git push origin ale
Fai un commit ogni volta che completi una modifica logica (es. cambiato una sezione, aggiunto una foto, ecc.).

ComandoCosa fa

git statusVedi cosa è stato modificato

5 — Regole d'oro

1

Lavora sempre su ale, mai direttamente su main.

2

Prima di iniziare, aggiorna sempre il branch con git pull + git merge main.

3

Fai commit frequenti con descrizioni chiare — è più facile tornare indietro se qualcosa va storto.

4

Se hai dubbi su cosa sta succedendo, usa git status — ti dice sempre in che stato sei.

⚠️ Non fare mai git push --force senza chiedere — può sovrascrivere il lavoro degli altri.
